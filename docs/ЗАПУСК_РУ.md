# Как запустить Flux у себя на компьютере

## 1. Что должно быть установлено

- **Node.js** (желательно версия 18 или новее)
- **Microsoft SQL Server** (локально или в облаке) — база нужна для входа и регистрации

## 2. Политика PowerShell и команда `npm`

Если при `npm install` пишет про запрет сценариев:

- откройте **cmd** (не PowerShell) и выполняйте команды там, **или**
- в PowerShell один раз выполните:  
  `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`  
  **или** вызывайте `npm.cmd install`

## 3. Установка пакетов

```bat
cd C:\Users\zzki\flux-store
npm install
```

После установки для API выполнится `prisma generate` (создаётся клиент к базе).

## 4. Настройка базы данных

1. Создайте файл `apps\api\.env` по образцу `apps\api\.env.example`.
2. Укажите **DATABASE_URL** — строку подключения к SQL Server (логин, пароль, имя базы `flux_store` и т.д.).
3. Примените схему и тестовые данные:

```bat
cd C:\Users\zzki\flux-store
npx prisma db push -w @flux/api
npm run seed
```

## 6. Запуск сайта и сервера

В одном окне терминала:

```bat
npm run dev -w @flux/web
```

В другом:

```bat
npm run dev -w @flux/api
```

- Сайт: http://localhost:3000  
- Проверка API: http://localhost:4000/api/v1/health  

В файле `apps\web\.env.local` (можно скопировать из `.env.example`) укажите при необходимости:

`NEXT_PUBLIC_API_URL=http://localhost:4000/api/v1`

## 7. Если вход не работает

- Убедитесь, что **SQL Server запущен** и строка `DATABASE_URL` верная.
- Убедитесь, что выполнены `db push` и `seed`.
- Откройте консоль браузера (F12) — если ошибка «сеть», проверьте, что API запущен на порту 4000.
